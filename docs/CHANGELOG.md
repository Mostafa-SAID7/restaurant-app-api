# 📋 Changelog

All notable changes to **Restaurant API** will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Added
- Repository pattern implementation with Unit of Work
- Comprehensive service layer architecture
- AutoMapper integration for object mapping
- Swagger API documentation
- Image upload functionality (file and base64)
- Shopping cart management system
- Order processing with master orders
- User authentication with API keys
- Angular 19 frontend application
- Responsive UI components
- Clean architecture implementation

### Changed
- Migrated from direct DbContext usage to Repository pattern
- Improved error handling and logging
- Enhanced API documentation with Swagger annotations
- Refactored services to use dependency injection
- Updated project structure for better maintainability

### Fixed
- Resolved duplicate method implementations
- Fixed compilation errors and warnings
- Improved null reference handling
- Enhanced data validation

## [1.0.0] - 2026-03-18

### Added
- Initial project setup
- Basic CRUD operations for restaurants
- User management system
- Order processing functionality
- Database migrations and models
- API endpoints for all core features

### Security
- API key-based authentication
- Input validation and sanitization
- SQL injection prevention through EF Core
- CORS configuration for secure cross-origin requests
